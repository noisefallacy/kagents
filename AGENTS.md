# Repository Working Notes

## Purpose

This project is both:

- a custom agent application built on Google ADK first
- a learning project for understanding Google ADK design, tooling, and patterns

## Implementation Journal Rule

When making a meaningful change, update the implementation journal in
`docs/implementation_journal.md` within the same task.

Meaningful changes include:

- adding or removing an agent, tool, or capability
- changing architecture or routing behavior
- introducing new setup or environment conventions
- adding an external integration such as MCP, API clients, or web tools
- fixing a bug that changes runtime behavior in a non-trivial way

Journal entries should stay concise and practical. Include:

- date
- change summary
- what was learned about ADK
- follow-up notes if useful

If a task is purely cosmetic or trivial, a journal update is optional.
